# 驱动管理

本目录用于存放各种硬件设备的驱动文件，包括墨水屏、传感器、音频模块等。

## 驱动类型

### 墨水屏驱动
- 支持多种尺寸和颜色的墨水屏
- 支持电子价签和二手阅读器屏幕
- 驱动文件应包含初始化、绘制、更新等核心功能

### 传感器驱动
- 温湿度传感器
- 人体感应传感器
- 光照传感器

### 音频驱动
- 音频编解码器驱动
- 音频放大器驱动

### 触摸驱动
- 电容触摸屏驱动
- 电阻触摸屏驱动

### 摄像头驱动
- 各种分辨率的摄像头驱动

## 驱动文件命名规范

```
[硬件类型]_[品牌]_[型号].[cpp/h]
例如：
eink_gooddisplay_gdew042z15.cpp
eink_gooddisplay_gdew042z15.h
sensor_aosong_dht22.cpp
sensor_aosong_dht22.h
```

## 驱动开发规范

1. 每个驱动应包含对应的头文件和源文件
2. 驱动应实现统一的接口，便于主程序调用
3. 驱动应包含初始化、配置、读取/写入、关闭等功能
4. 驱动应包含错误处理和状态反馈
5. 驱动应遵循低功耗设计原则

## 驱动使用方法

1. 将驱动文件复制到本目录
2. 在主程序中包含驱动头文件
3. 按照驱动文档初始化和使用驱动
4. 在config.h中配置对应的硬件型号

1. 驱动文件应与硬件型号严格对应
2. 不同型号的硬件可能需要不同的驱动参数
3. 驱动更新时应注意兼容性问题
4. 使用前请仔细阅读驱动文档和硬件 datasheet
